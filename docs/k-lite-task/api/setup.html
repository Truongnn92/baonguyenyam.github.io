Cài đặt

Trước khi sử dụng bộ kit này bạn phải hiểu rằng đây là bộ mã nguồn dùng để tham khảo, chúng tôi khước từ mọi trách nhiệm khi bạn gặp phải rắc rối về pháp lý hay sử dụng nó để thương mại hoá nó. Bằng cách sử dụng và tuân thủ luật nguồn mở nghĩa là bạn đã chấp nhận sự rủi ro khi sử dụng bộ kit này. Tuy nhiên chúng tôi luôn hỗ trợ bạn fix lỗi trong khả năng và thời gian cho phép.

Để cài đặt bộ kit này trước tiên bạn clone source về từ:

git clone https://github.com/baonguyenyam/k-lite-task.git

Sau đó tiến hành cài đặt bằng cách nhấp vào file

install.bat

Nếu bạn sử dụng Mac thì có thể chạy file install.sh

1) config.yml

Vào thư mục k-task mở fileconfig.yml để cấu hình các thông số cần thiết trong mục SEO

Tại đây bạn sẽ thấy phần ACTIVE_CONCAT: TRUE

a) Nếu giá trị là TRUE nghĩa là khi build sẽ không nén js/css trong thư mục pages mà sẽ được load dynamic theo từng page. Khi đó:

b) Trong file src/styles/main.sass bạn phải xoá đi phần các file sass được import từ thư mục pages

Trong file này cũng cho phép bạn gọi các thư viện từ bower_components vào bằng cách cấu hình mục concat => js và concat => css

2) config.js

Vào thư mục src/scripts để cấu hình các tính năng cho hệ thống, phần này có thể xem chi tiết tại đây

3) main.sass

Bộ kit này chỉ sử dụng viết SASS mà không viết SCSS vì vậy bạn phải tập làm quen nếu chưa dùng. Vào thư mục src/styles mở file main.sass trong file này bạn có thể import các file mình viết trogn thư mục pages vào

4) PUG

Bộ kit này cũng không viết HTML mà sử dụng template PUG làm nền tảng vì vậy bạn phải tập làm quen nếu chưa dùng. Mọi cấu hình được đặt trong file tempaltes/_layout/_config_menu.pug

Build

Để chạy Dev bạn chạy lệnh

npm start

Để chạy Build deploy lên server bạn chạy lệnh

npm run pro

Để chạy Build test offline bạn chạy lệnh

npm run prod

Để chạy Build KHÔNG NÉN deploy lên server bạn chạy lệnh

npm run pro-no

Để chạy Build KHÔNG NÉN test offline bạn chạy lệnh

npm run prod-no

Nếu bạn sử dụng mac hay Ubuntu bạn có thể gọi trực tiếp file app.sh

sh app.sh
